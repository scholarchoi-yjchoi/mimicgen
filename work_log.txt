- Isaac Lab MimicGen Blueprint 노트북의 "Data Generation" 셀이 블로킹됨

## 본 세션 작업 내용

### 1. 근본 원인 분석
- Jupyter와 Isaac Sim(Kit)이 각각 자체 asyncio 이벤트 루프를 사용
- `asyncio.get_event_loop()`가 Kit의 루프가 아닌 새 루프를 생성
- Kit의 async_engine에서 `assert events._get_running_loop() is self` 검증 실패

### 2. Standalone 스크립트 접근 (v9 계획)
- Jupyter 대신 독립 Python 스크립트로 데이터 생성 실행 결정
- 새 파일 생성: `notebook/generate_data_standalone.py`

### 3. 시도한 버전들

#### v1 (초기): 기본 asyncio 루프
- `asyncio.new_event_loop()` + `run_until_complete()` 사용
- 결과: `__aenter__` 오류 (asyncio_lock=None 문제)

#### v2: 더미 락 클래스 추가
- `DummyAsyncLock` 클래스 생성하여 `async with` 지원
- 결과: Kit async_engine의 AssertionError 지속

#### v3: 공식 Isaac Lab API 사용
- `setup_async_generation()`, `env_loop()` 함수 사용
- 결과: 여전히 AssertionError (새 루프와 Kit 루프 충돌)

#### v4: Kit 이벤트 루프 직접 사용 시도
- `omni.kit.async_engine.async_engine._loop` 접근 시도
- 결과: AttributeError (API가 다름)

#### v5 (성공): nest_asyncio + 공식 API
- `nest_asyncio.apply()` 추가 (노트북과 동일한 환경)
- 공식 API (`setup_async_generation()`, `env_loop()`) 사용
- Python `-u` 플래그로 unbuffered 출력
- 결과: **성공!**

### 4. 최종 해결책

#### 핵심 코드 (generate_data_standalone.py)
```python
# nest_asyncio 적용 (노트북과 동일하게)
import nest_asyncio
nest_asyncio.apply()

# ... AppLauncher 초기화 ...

# 공식 API 사용
from isaaclab_mimic.datagen.generation import setup_env_config, env_loop, setup_async_generation

async_gen = setup_async_generation(
    env=env,
    num_envs=args_cli.num_envs,
    input_file=args_cli.input_file,
    success_term=success_term,
    pause_subtask=args_cli.pause_subtask
)

env_loop(
    env=env,
    env_action_queue=async_gen['action_queue'],
    shared_datagen_info_pool=async_gen['info_pool'],
    asyncio_event_loop=async_gen['event_loop']
)
```

#### 실행 명령
```bash
docker exec synthetic-manipulation-motion-generation-isaac-lab-1 \
    /workspace/isaaclab/_isaac_sim/python.sh -u \
    /workspace/isaaclab/generate_data_standalone.py
```

- 컨테이너 무한 재시작 문제 해결

### 문제 상황

#### 문제 1: 출력 데이터 손실
- 기존: 입력 파일만 마운트 (`samples/annotated_dataset.hdf5`)
- 출력 파일(`generated_dataset.hdf5`)과 이미지(`output/`)가 컨테이너 내부에만 저장
- 컨테이너 종료 시 생성된 데이터 손실

#### 문제 2: 컨테이너 무한 재시작
- `restart: unless-stopped` 설정으로 작업 완료 후에도 재시작

#### 문제 3: 생성 횟수 고정
- `generation_num_trials: 1` 하드코딩되어 변경 불가

### 해결 방법

#### 1. 로컬 폴더 생성 및 입력 파일 복사
```bash
mkdir -p datasets output
cp samples/annotated_dataset.hdf5 datasets/
```

#### 2. docker-compose.yml 수정

**환경 변수 추가:**
```yaml
environment:
  MODE: ${MODE:-jupyter}
  NUM_TRIALS: ${NUM_TRIALS:-1}
```

**볼륨 마운트 변경:**
```yaml
volumes:
  - ./datasets:/workspace/isaaclab/datasets    # 폴더 전체 마운트
  - ./output:/workspace/isaaclab/output        # 출력 이미지 마운트 추가
```

#### 3. generate_data_standalone.py 수정

```python
# 환경 변수에서 생성 횟수 읽기 (기본값: 1)
num_trials = int(os.environ.get("NUM_TRIALS", "1"))
print(f"[설정] 목표 생성 횟수: {num_trials}", flush=True)

config = {
    "generation_num_trials": num_trials,
    ...
}
```

#### 4. launch.sh 수정

```bash
# 필수 패키지 설치 추가
/isaac-sim/kit/python/bin/python3 -m pip install nest_asyncio toml -q
```

### 실행 방법

```bash

# 여러 번 생성 (예: 5회)
```

## 세션 6: Jupyter 노트북 Data Generation 셀 수정

### 작업 일시
2026-02-04

### 목적
- Jupyter 노트북의 Data Generation 셀을 검증된 방식(v9)으로 수정
- standalone 스크립트에서 성공한 공식 API 방식 적용

### 문제 상황
- 기존 v8 버전: 별도 스레드에서 `DataGenerator.generate()` 직접 호출

## 세션 9: subprocess 방식 개선 (v11)

### 작업 일시
2026-02-04

### 문제 상황
- v10에서 `simulation_app.close()` 후 subprocess가 블로킹됨
- 출력이 이전 셀에 나타나는 문제
- GPU 리소스 충돌 가능성

### 근본 원인
- 동일 프로세스에서 Isaac Sim 종료 후 subprocess로 새 Isaac Sim 시작 시 충돌
- GPU 리소스가 완전히 해제되지 않은 상태에서 새 인스턴스 시작

### 해결 방법 (v11)

**핵심 변경:**
1. `shell=True` - shell script 실행 문제 해결
2. `start_new_session=True` - 별도 프로세스 그룹에서 실행
3. `time.sleep(3)` - GPU 리소스 해제 대기
4. `sys.stdout.flush()` - 출력 버퍼 강제 플러시
5. `NameError` 처리 - simulation_app 없이도 작동

**수정 파일:** `notebook/generate_dataset.ipynb` - Data Generation 셀

```python
# v11: subprocess 전용 Data Generation
import subprocess
import os
import sys

# 기존 simulation_app이 있으면 종료 시도 (없어도 OK)
try:
    simulation_app.close()
    print("[INFO] 기존 simulation_app 종료됨")
    import time
    time.sleep(3)  # GPU 리소스 해제 대기
except NameError:
    print("[INFO] simulation_app 없음 - subprocess만 실행")

# 환경 변수 설정
env_vars = os.environ.copy()
env_vars["NUM_TRIALS"] = str(num_trials.value)

sys.stdout.flush()

# subprocess로 standalone 스크립트 실행
process = subprocess.Popen(
    "/workspace/isaaclab/_isaac_sim/python.sh -u /workspace/isaaclab/generate_data_standalone.py",
    shell=True,
    stdout=subprocess.PIPE,
    stderr=subprocess.STDOUT,
    text=True,
    bufsize=1,
    env=env_vars,
    start_new_session=True
)

# 실시간 출력 표시
for line in iter(process.stdout.readline, ''):
    print(line, end='', flush=True)
```

## 세션 11: v13 - SKIP_SIMULATION + os.system 방식

### 작업 일시
2026-02-04

### 문제 상황
- v10~v12 (subprocess.Popen, IPython !) 모두 동일한 문제 발생
- subprocess가 블로킹되고 출력이 "Spin up the Simulation" 셀에 나타남
- Data Generation 셀 자체는 `[*]` 상태로 무한 대기

### 근본 원인 분석 (웹 검색 결과)

#### 공식 문서에서 확인된 사항
1. **"콜백 내부의 인쇄는 노트북에 표시되지 않고 Omniverse 터미널에 나타납니다"**
   - [Isaac Sim Jupyter Notebook Documentation](https://docs.isaacsim.omniverse.nvidia.com/latest/development_tools/jupyter_notebook.html)
2. **IPython 매직 명령(`!`)은 지원되지 않음**
3. **"차단되는 코드 실행은 Isaac Sim을 정지시킵니다"**

#### NVIDIA 포럼에서 확인된 사항
- **별도 Python 셸에서 실행 중인 Isaac Sim 인스턴스로 연결할 수 없습니다**
  - [NVIDIA Developer Forums](https://forums.developer.nvidia.com/t/isaac-sim-from-jupyter-notebook/195125)
- subprocess로 별도 프로세스 실행 시 stdout이 원래 Isaac Sim 셀에 연결됨

#### 핵심 결론
**Isaac Sim이 시작되면 해당 셀의 출력 영역에 stdout이 고정됨.**
이후 셀에서 어떤 방법(subprocess, IPython !, os.system 등)을 사용해도 출력이 첫 번째 셀("Spin up the Simulation")에 나타남.

### 해결 방법 (v13)

#### 핵심 아이디어
- **Isaac Sim을 Jupyter에서 시작하지 않음**
- "Spin up the Simulation" 셀을 스킵
- Data Generation 셀에서 standalone 스크립트만 실행 (별도 프로세스)
- stdout 리다이렉션 문제 원천 차단

#### 노트북 흐름 변경

**기존 (문제 발생):**
```
[Setup] → [Spin up Simulation] → [Parameters] → [Data Generation]
              ↓ Isaac Sim 시작
              ↓ stdout 고정됨
              ↓ 이후 모든 출력이 여기에 나타남
```

**변경 후 (해결):**
```
[Setup] → [Spin up Simulation: SKIP] → [Parameters: SKIP] → [Data Generation]
                                                                    ↓
                                                        standalone 스크립트 실행
                                                        (새로운 Isaac Sim)
                                                                    ↓
                                                        출력이 이 셀에 표시됨
```

### 수정 내용

#### 1. "Spin up the Simulation" 셀 수정
